# AI-Generated UI Patterns to Avoid

AIがUIを生成すると特定のパターンに収束する。これらを意識的に回避することで、人間がデザインした質の高いUIに近づける。

---

## 🎨 カラー

| ❌ 避けるパターン | ✅ 代替案 |
|-----------------|---------|
| 紫〜青のグラデーション（indigo-500, violet系） | ブランドカラーから派生した独自パレット |
| 白背景 + 紫アクセント | コンテキストに合った色選び |
| ダークモード + ネオンアクセント | 落ち着いたダークテーマ or ライトテーマ |
| Tailwindデフォルトのカラーパレット | カスタムカラーパレットを定義 |
| 控えめすぎる「安全な」色選び | 大胆なアクセントカラー |

**チェックポイント:**
- [ ] Primary colorがindigo/violet/purpleではない
- [ ] グラデーションが必要な理由を説明できる
- [ ] カラーパレットはブランドから導出されている

---

## 📐 レイアウト

| ❌ 避けるパターン | ✅ 代替案 |
|-----------------|---------|
| 3カラムのアイコン付きカード | 用途に応じた適切なレイアウト |
| 何でもCardに入れる | 必要な場合のみContainer/Surfaceを使用 |
| 左テキスト + 右画像のヒーロー | 非対称、フルブリード、オーバーラップ |
| 均等なグリッド配置 | 視覚的階層を意識した配置 |
| 過剰な要素数 | ミニマルに必要なものだけ |

**チェックポイント:**
- [ ] 各Cardに存在理由がある
- [ ] 3カラムグリッドを無意識に使っていない
- [ ] 視覚的階層が明確

---

## 🔤 タイポグラフィ

| ❌ 避けるパターン | ✅ 代替案 |
|-----------------|---------|
| Inter, Roboto, Arial | 個性的なフォント選択 |
| システムフォントのまま | Google Fontsから適切なフォントを選択 |
| Space Grotesk（AI代替デフォルト） | コンテキストに合ったフォント |
| 見出しと本文のコントラスト不足 | 明確なフォント階層 |

**推奨フォント例（日本語対応）:**
- Noto Sans JP + 英字: DM Sans, Outfit, Sora
- Zen Kaku Gothic New
- LINE Seed JP
- M PLUS Rounded 1c（カジュアル）

**チェックポイント:**
- [ ] フォントファミリーを意図的に選択した
- [ ] Weight階層が定義されている（Regular, Medium, Bold等）
- [ ] 日本語と英字の組み合わせを考慮

---

## 🖼️ ビジュアル要素

| ❌ 避けるパターン | ✅ 代替案 |
|-----------------|---------|
| アウトラインアイコンのみ | Filled、Duotone、カスタムアイコン |
| ✨スパークル/キラキラ多用 | 意味のあるアイコン |
| 角丸8px/16px固定 | コンテキストに応じた角丸 |
| グラスモーフィズムの乱用 | 必要な場合のみ効果を使用 |
| ストックフォト的な汎用画像 | カスタムイラスト、実際の写真 |

**チェックポイント:**
- [ ] アイコンスタイルが統一されている
- [ ] 角丸の値に一貫性がある（4, 8, 12, 16, 24など）
- [ ] 装飾的要素に意味がある

---

## 🏗️ コンポーネント

| ❌ 避けるパターン | ✅ 代替案 |
|-----------------|---------|
| FABをとりあえず配置 | Primary actionが明確な場合のみ |
| 巨大なCTAボタン + グラデーション | 適切なサイズ、ソリッドカラー |
| 無意味なプログレスリング | 意味のある進捗表示 |
| Material 3デフォルトそのまま | カスタムデザインシステム |
| チャットUIの乱用 | 適切なインタラクションパターン |

**チェックポイント:**
- [ ] 各コンポーネントの存在理由を説明できる
- [ ] Material 3のデフォルトをそのまま使っていない
- [ ] ボタンの視覚的階層が明確（Primary > Secondary > Tertiary）

---

## 🎭 全体的な印象

| ❌ 避けるパターン | ✅ 代替案 |
|-----------------|---------|
| 「モダン」「クリーン」「ミニマル」で片付ける | 具体的なデザイン方向性を定義 |
| どのアプリも同じ「Tech Startup」感 | 独自のブランドアイデンティティ |
| 2020年頃のトレンドに固執 | 最新のUIトレンドを参考 |
| コンテキストを無視した汎用デザイン | ユーザー・用途に最適化 |

---

## Flutter固有の回避パターン

| ❌ 避けるパターン | ✅ 代替案 |
|-----------------|---------|
| ThemeData.light()そのまま | カスタムThemeData |
| ColorScheme.fromSeed() | 手動でカラーパレット定義 |
| useMaterial3: true のデフォルト | カスタムコンポーネント |
| AppBar + Scaffold定型 | カスタムレイアウト検討 |
| ListView.builder安易多用 | CustomScrollView + Slivers |

---

## コードレビュー時のチェック

```dart
// ❌ AIっぽいコード
ThemeData(
  colorScheme: ColorScheme.fromSeed(seedColor: Colors.deepPurple),
  useMaterial3: true,
)

// ✅ カスタムテーマ
ThemeData(
  colorScheme: ColorScheme.light(
    primary: Color(0xFF1A1A2E),      // 独自のプライマリ
    secondary: Color(0xFFE94560),     // 独自のセカンダリ
    surface: Color(0xFFF5F5F7),       // 独自のサーフェス
  ),
  textTheme: GoogleFonts.dmSansTextTheme(), // 独自フォント
)
```

## 参考：良いUIの特徴

- **意図が明確** — 各要素に存在理由がある
- **一貫性** — カラー、タイポ、スペーシングが統一
- **差別化** — 他のアプリと区別できる個性
- **コンテキスト適合** — ユーザー・用途に最適化
- **心地よさ** — 触っていて気持ちいいインタラクション
