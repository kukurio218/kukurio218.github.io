# Pre-Delivery Checklist for Flutter Apps

UIを納品/リリースする前の最終チェックリスト。全項目をクリアしてから納品する。

---

## 🎨 ビジュアル品質

### カラー
- [ ] Primary/Secondary/Accentカラーが定義され、一貫して使用されている
- [ ] テキストコントラスト比が4.5:1以上（WCAG AA）
- [ ] ダークモード対応している（または意図的に非対応と決定）
- [ ] エラー/成功/警告の色が直感的（赤/緑/オレンジ）
- [ ] AIっぽい紫グラデーションを使っていない

### タイポグラフィ
- [ ] フォントファミリーが意図的に選択されている（Inter/Robotoデフォルトではない）
- [ ] フォント階層が明確（Display > H1 > H2 > Body > Caption）
- [ ] 行間（height）が適切（1.4〜1.6）
- [ ] 日本語フォントが適切に設定されている（Noto Sans JP等）
- [ ] 最小フォントサイズが12px以上

### スペーシング
- [ ] 一貫したスペーシングシステム（4, 8, 16, 24, 32等）
- [ ] 要素間の余白が適切（詰まりすぎ/空きすぎがない）
- [ ] 画面端のパディングが統一されている

### アイコン
- [ ] アイコンスタイルが統一されている（Outlined/Filled/Duotone）
- [ ] 絵文字をアイコン代わりに使っていない（SVGアイコンを使用）
- [ ] アイコンサイズが一貫している

### 画像
- [ ] 画像のアスペクト比が統一されている
- [ ] プレースホルダー/ローディング状態がある
- [ ] エラー時の代替画像がある
- [ ] 画像が適切にキャッシュされている

---

## 🖱️ インタラクション

### タップ/クリック
- [ ] 全てのタップ可能な要素にフィードバックがある
- [ ] タップターゲットが最小44x44pt（推奨48x48pt）
- [ ] ボタンに`cursor: pointer`相当の視覚的手がかり
- [ ] ダブルタップ防止（連打でエラーにならない）

### ホバー/フォーカス（Web/Desktop）
- [ ] ホバー状態が定義されている
- [ ] フォーカス状態が視覚的に明確
- [ ] トランジションが滑らか（150-300ms）

### アニメーション
- [ ] 意味のあるアニメーションのみ使用
- [ ] アニメーション時間が適切（200-400ms）
- [ ] `prefers-reduced-motion`を尊重
- [ ] 無限ループアニメーションは必要最小限

### 触覚フィードバック
- [ ] 重要なアクションにHapticFeedbackを使用
- [ ] フィードバックの強度が適切（Light/Medium/Heavy）

---

## 📱 レスポンシブ/マルチデバイス

### 画面サイズ
- [ ] iPhone SE (375px) で崩れない
- [ ] iPhone 14 Pro Max (430px) で崩れない
- [ ] iPad (768px) で適切に表示される
- [ ] Androidの様々な画面サイズで確認

### 向き
- [ ] Portrait（縦）で正しく表示
- [ ] Landscape（横）で正しく表示、または適切にロック

### セーフエリア
- [ ] ノッチ/Dynamic Islandを考慮
- [ ] ホームインジケーター領域を考慮
- [ ] `SafeArea`が適切に使用されている

### テキストスケール
- [ ] システムフォントサイズ変更でレイアウトが崩れない
- [ ] 200%拡大でも使用可能

---

## ♿ アクセシビリティ

### スクリーンリーダー
- [ ] 全てのインタラクティブ要素にセマンティクスラベル
- [ ] 画像に代替テキスト
- [ ] 読み上げ順序が論理的
- [ ] 装飾的要素は`excludeFromSemantics`

### 色
- [ ] 色だけに依存した情報伝達をしていない
- [ ] 色覚異常でも区別できる

### 動き
- [ ] 前庭障害への配慮（激しい動きを避ける）
- [ ] 点滅は3回/秒以下

### 入力
- [ ] キーボード操作のみで使用可能（Web）
- [ ] フォーカストラップなし

---

## ⚡ パフォーマンス

### ビルド最適化
- [ ] `const`を可能な限り使用
- [ ] 不要な再ビルドを避ける（`Consumer`の適切な使用）
- [ ] 画像サイズが最適化されている

### リスト
- [ ] 長いリストは`ListView.builder`を使用
- [ ] 適切な`key`が設定されている
- [ ] 画面外の要素は描画されない

### ネットワーク
- [ ] ローディング状態が表示される
- [ ] エラー状態が適切にハンドリングされる
- [ ] オフライン状態への対応

### 起動時間
- [ ] 起動画面（Splash）が適切
- [ ] 重い処理を起動時に行わない

---

## 🔒 UX/安全性

### フォーム
- [ ] 入力バリデーションがリアルタイム
- [ ] エラーメッセージが具体的で解決策を示す
- [ ] 必須フィールドが明確
- [ ] 適切なキーボードタイプ（email, phone等）
- [ ] オートフィルがサポートされている

### ナビゲーション
- [ ] 戻るボタンが期待通りに動作
- [ ] 現在地が明確（どの画面にいるか）
- [ ] デッドエンドがない

### 状態
- [ ] 空の状態（Empty State）がデザインされている
- [ ] ローディング状態がある
- [ ] エラー状態がある
- [ ] 成功状態のフィードバックがある

### 破壊的アクション
- [ ] 削除等の前に確認ダイアログ
- [ ] 取り消し（Undo）可能な場合はその機能を提供

---

## 🧪 テスト確認

### 実機テスト
- [ ] iOS実機で確認
- [ ] Android実機で確認
- [ ] 低スペック端末で確認

### エッジケース
- [ ] 長いテキストで崩れない（overflow対応）
- [ ] 空データでクラッシュしない
- [ ] 特殊文字で問題ない
- [ ] RTL言語で崩れない（必要な場合）

---

## 📋 クイックチェック（最低限）

納品前に最低限確認すべき10項目:

1. [ ] タップ可能な要素は全てフィードバックがある
2. [ ] テキストコントラストが十分
3. [ ] ローディング/エラー/空の状態がある
4. [ ] iPhone SE (375px) で崩れない
5. [ ] セーフエリアを考慮している
6. [ ] フォントが意図的に選択されている
7. [ ] カラーパレットが一貫している
8. [ ] 絵文字をアイコン代わりに使っていない
9. [ ] アニメーションが滑らか（150-300ms）
10. [ ] 戻るボタンが正しく動作する

## Flutter コードチェック

```dart
// ❌ 避けるべきパターン

// 1. ハードコードされた色
color: Color(0xFF6200EE), // 散在

// 2. マジックナンバー
padding: EdgeInsets.all(17), // なぜ17?

// 3. デフォルトテーマそのまま
ThemeData(
  colorScheme: ColorScheme.fromSeed(seedColor: Colors.deepPurple),
  useMaterial3: true,
)

// 4. constなしのWidget
Container(
  padding: EdgeInsets.all(16), // constにできる
)

// 5. セマンティクスなし
GestureDetector(
  onTap: onTap,
  child: Icon(Icons.delete),
)


// ✅ 推奨パターン

// 1. カラーは定数で管理
color: AppColors.primary,

// 2. スペーシングは定数で管理
padding: AppSpacing.allMd,

// 3. カスタムテーマ
ThemeData(
  colorScheme: ColorScheme.light(
    primary: AppColors.primary,
    secondary: AppColors.secondary,
  ),
  textTheme: AppTypography.textTheme,
)

// 4. constを使用
const SizedBox(height: 16),

// 5. セマンティクスを追加
Semantics(
  button: true,
  label: '削除',
  child: GestureDetector(
    onTap: onTap,
    child: Icon(Icons.delete),
  ),
)
```
