## ソーシャル / マッチング

**代表アプリ:** Tinder, Bumble, Hinge, Instagram, TikTok

### 特徴的なUIパターン
- **カードスワイプ**: 左右のスワイプで意思決定
- **フルスクリーン写真/動画**: ビジュアル重視
- **ストーリー形式**: 24時間で消える一時的コンテンツ
- **無限スクロール**: フィードの連続的な閲覧

### 重要な画面
| 画面 | ポイント |
|------|---------|
| フィード/発見 | コンテンツファースト、ナビは最小限 |
| プロフィール | 写真が主役、情報は段階的に開示 |
| チャット | リアルタイム感、既読表示 |
| マッチ/通知 | 感情的なフィードバック |

### 設計のコツ
- 写真/動画のクオリティを最大化
- アクションは最小限（いいね、コメント、シェア）
- 通知で再訪を促す
- プライバシー設定を分かりやすく

### 参考になるUI要素
```
- スワイプカード（Tinder）
- ストーリーリング（Instagram）
- リアクションアニメーション（Facebook）
- プロフィールバッジ（Bumble）
```

## Eコマース / ショッピング

**代表アプリ:** Amazon, ZOZOTOWN, Shopify, SHEIN

### 特徴的なUIパターン
- **商品グリッド**: 2-3カラムのグリッド表示
- **カルーセル**: おすすめ、セール、カテゴリ
- **フィルター/ソート**: 細かい絞り込み
- **カート**: 常にアクセス可能

### 重要な画面
| 画面 | ポイント |
|------|---------|
| 商品一覧 | 素早いスキャン、比較のしやすさ |
| 商品詳細 | 写真ギャラリー、レビュー、在庫 |
| カート | 編集しやすさ、合計金額の明確さ |
| チェックアウト | ステップの可視化、入力の最小化 |

### 設計のコツ
- 商品写真は統一されたアスペクト比
- 価格、割引を目立たせる
- ワンタップでカート追加
- ゲストチェックアウト対応

### 参考になるUI要素
```
- 商品カード（価格、評価、割引バッジ）
- サイズ/カラー選択UI
- レビュー星評価
- 「お気に入りに追加」ハートアイコン
```

## フィンテック / 金融

**代表アプリ:** PayPay, Revolut, Robinhood, Coinbase

### 特徴的なUIパターン
- **残高表示**: 大きく目立つ数字
- **取引履歴**: 時系列リスト
- **クイックアクション**: 送金、入金、支払い
- **グラフ/チャート**: 資産推移の可視化

### 重要な画面
| 画面 | ポイント |
|------|---------|
| ホーム/残高 | 最重要情報を大きく、セキュリティ感 |
| 取引詳細 | 明細の透明性 |
| 送金/支払い | 金額入力のしやすさ、確認画面 |
| 本人確認 | 信頼感、進捗の可視化 |

### 設計のコツ
- 数字は見やすいフォント（等幅推奨）
- セキュリティを感じさせる色使い（青、緑）
- エラーは即座に明確に表示
- 生体認証対応

### 参考になるUI要素
```
- 残高カード（隠す/表示切り替え）
- 取引リスト（アイコン、金額、日時）
- 数字キーパッド
- 資産推移グラフ
```

## ヘルスケア / フィットネス

**代表アプリ:** Nike Run Club, Strava, MyFitnessPal, Headspace

### 特徴的なUIパターン
- **ダッシュボード**: 今日の目標、進捗
- **リング/プログレス**: 達成度の可視化
- **カレンダービュー**: 履歴の確認
- **ワークアウト画面**: 大きな数字、シンプルな操作

### 重要な画面
| 画面 | ポイント |
|------|---------|
| ホーム/今日 | モチベーションを高める表示 |
| ワークアウト中 | 大きな数字、最小限のUI |
| 履歴/統計 | 成長の可視化、達成感 |
| ソーシャル | 他者との比較、応援 |

### 設計のコツ
- 達成時の演出（アニメーション、触覚）
- ダークモード対応（夜間利用）
- 音声フィードバック対応
- Apple Health / Google Fit連携

### 参考になるUI要素
```
- アクティビティリング（Apple Watch式）
- ワークアウトカード
- ストリークカウンター
- 達成バッジ/トロフィー
```

## 生産性 / タスク管理

**代表アプリ:** Notion, Todoist, Things 3, Linear

### 特徴的なUIパターン
- **リスト**: チェックボックス付きタスク
- **カンバン**: ステータス別カラム
- **カレンダー**: スケジュール表示
- **クイック入力**: どこからでもタスク追加

### 重要な画面
| 画面 | ポイント |
|------|---------|
| タスクリスト | スキャンしやすさ、優先度の明確さ |
| タスク詳細 | 編集のしやすさ、サブタスク |
| 今日/フォーカス | 集中すべきものだけ表示 |
| 設定/統合 | 外部サービス連携 |

### 設計のコツ
- キーボードショートカット対応
- ドラッグ&ドロップ
- 完了時の心地よいフィードバック
- シンプルさを維持

### 参考になるUI要素
```
- チェックボックス（アニメーション付き）
- 優先度インジケーター（色、アイコン）
- 日付ピッカー
- タグ/ラベル
```

## メディア / エンターテインメント

**代表アプリ:** Netflix, Spotify, YouTube, Apple Music

### 特徴的なUIパターン
- **フルスクリーンプレイヤー**: 没入感
- **横スクロールカルーセル**: カテゴリ別コンテンツ
- **パーソナライズドホーム**: AIによるおすすめ
- **オフライン対応**: ダウンロード機能

### 重要な画面
| 画面 | ポイント |
|------|---------|
| ホーム/発見 | パーソナライズされたおすすめ |
| プレイヤー | シンプルなコントロール、没入感 |
| 検索 | 音声検索、フィルター |
| ライブラリ | 整理、プレイリスト |

### 設計のコツ
- コンテンツを邪魔しないUI
- ダークテーマがデフォルト
- Picture-in-Picture対応
- キャスト/AirPlay対応

### 参考になるUI要素
```
- コンテンツカード（サムネイル、タイトル、進捗バー）
- ミニプレイヤー
- シークバー
- 再生キュー
```

## フード / デリバリー

**代表アプリ:** Uber Eats, 出前館, DoorDash, Instacart

### 特徴的なUIパターン
- **店舗リスト**: 評価、配達時間、写真
- **メニュー**: カテゴリ別、写真付き
- **カート**: フローティング表示
- **トラッキング**: リアルタイム配達状況

### 重要な画面
| 画面 | ポイント |
|------|---------|
| 店舗一覧 | 写真、評価、配達時間が一目で |
| 店舗詳細/メニュー | カテゴリナビ、カスタマイズ |
| カート/チェックアウト | 合計金額、配達先確認 |
| 配達トラッキング | 地図、ステータス、連絡手段 |

### 設計のコツ
- 食欲をそそる写真表示
- 配達時間の目立つ表示
- 簡単な再注文機能
- ドライバーとの連絡手段

### 参考になるUI要素
```
- 店舗カード（写真、評価、配達時間、料金）
- メニューアイテム（写真、説明、価格）
- フローティングカートバー
- 配達ステータスタイムライン
```

## トラベル / 予約

**代表アプリ:** Airbnb, Booking.com, Google Maps, Expedia

### 特徴的なUIパターン
- **検索フォーム**: 日付、人数、場所
- **地図ビュー**: 位置ベースの探索
- **フィルター**: 価格、評価、アメニティ
- **予約フロー**: ステップバイステップ

### 重要な画面
| 画面 | ポイント |
|------|---------|
| 検索/発見 | 直感的な検索、インスピレーション |
| リスト/地図 | 切り替え可能、比較しやすさ |
| 詳細 | 写真ギャラリー、レビュー、設備 |
| 予約確認 | 明確な料金内訳、キャンセルポリシー |

### 設計のコツ
- 日付選択のしやすさ（カレンダー）
- 写真は大きく、たくさん
- レビューの信頼性を示す
- オフライン予約確認対応

### 参考になるUI要素
```
- 日付範囲ピッカー
- 地図上のピン（価格表示）
- 写真ギャラリー（スワイプ）
- アメニティアイコンリスト
```

## 教育 / 学習

**代表アプリ:** Duolingo, Coursera, Notion, Anki

### 特徴的なUIパターン
- **レッスン形式**: ステップバイステップ
- **ゲーミフィケーション**: ポイント、ストリーク、バッジ
- **進捗トラッキング**: 完了率、スキルツリー
- **復習システム**: 間隔反復

### 重要な画面
| 画面 | ポイント |
|------|---------|
| ホーム/コース | 今日のタスク、進捗 |
| レッスン | 集中できるシンプルなUI |
| 復習/テスト | フィードバック、正解率 |
| 実績/統計 | モチベーション維持 |

### 設計のコツ
- 小さな成功体験を頻繁に
- ストリーク（連続日数）で習慣化
- 間違いを恐れない雰囲気
- リマインダー通知

### 参考になるUI要素
```
- レッスンカード（進捗、所要時間）
- クイズUI（選択肢、入力）
- スキルツリー/マップ
- ストリークカウンター
```

## 共通パターン

### ナビゲーション
| パターン | 適したカテゴリ |
|---------|--------------|
| Bottom Tab (3-5) | ほぼ全て |
| Drawer | 設定が多いアプリ |
| Top Tab | コンテンツ切り替え |
| フルスクリーン + 戻る | メディア、マッチング |

### オンボーディング
| パターン | 適したカテゴリ |
|---------|--------------|
| カルーセル説明 | 初めて使うアプリ |
| プログレッシブ | フィンテック、ヘルスケア |
| チュートリアル | ゲーミフィケーション系 |
| 即座に体験 | ソーシャル、メディア |

### 検索
| パターン | 適したカテゴリ |
|---------|--------------|
| 検索バー + フィルター | Eコマース、トラベル |
| 音声検索 | メディア、ナビ |
| ビジュアル検索 | ショッピング |
| 最近の検索 + おすすめ | ほぼ全て |
